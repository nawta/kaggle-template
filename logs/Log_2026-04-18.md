# Log 2026-04-18

## グローバル MCP サーバー導入と review4 拡張

### 背景
- これまで `codex` MCP のみ project-local（`.claude.json` のローカルセクション）に登録されていた
- 全プロジェクトで使えるようにするため user スコープ（`~/.claude.json`）へ移行
- 加えて arxiv / GitHub / Kaggle / Deepwiki の各 MCP を user スコープで追加
- review4 スキルに CodeRabbit CLI と Deepwiki MCP の参照を組み込む

### 実施内容

#### MCP サーバー登録（user スコープ）
| 名前 | トランスポート | エンドポイント / コマンド | 認証 |
|------|--------------|------------------------|------|
| codex | stdio | `codex mcp-server` | — |
| arxiv | stdio | `uvx arxiv-mcp-server --storage-path ~/.arxiv-mcp-papers` | — |
| github | http | `https://api.githubcopilot.com/mcp/` | `gh auth token` 流用 |
| kaggle | http | `https://www.kaggle.com/mcp` | `~/.kaggle/kaggle.json` の Basic Auth |
| deepwiki | http | `https://mcp.deepwiki.com/mcp` | 不要 |

`claude mcp list` で5本すべて Connected を確認。

#### review4 スキル更新
- `scripts/review4.sh`:
  - `run_coderabbit()` 関数を追加し、5本目のレビュアーとして並列実行
  - `cr` 未インストール / 未ログインの場合はメッセージ付きでスキップ
  - last-commit スコープ時は `cr --base HEAD~1 --plain` を使用
  - タイムアウトは `CODERABBIT_TIMEOUT`（既定 1800 秒）で制御
- `SKILL.md`:
  - `allowed-tools` に Deepwiki MCP の3ツール（`read_wiki_structure` / `read_wiki_contents` / `ask_question`）を追加
  - 「事前コンテキスト補完」ステップで Deepwiki に外部 OSS 仕様の裏取りを委譲
  - 統合タグに `[CodeRabbit]` `[Deepwiki: <repo>]` を追加
- `docs/review4.md`:
  - 概要 / 動作フロー / レビュアー表 / 前提条件を 5 レビュアー版に更新
  - CodeRabbit タイムアウトのドキュメントを追加

### GitHub MCP の認証について
- 現状は `gh auth token`（OAuth `gho_...`）を流用
- 必要なスコープ（repo / read:org など）が満たされない場合は fine-grained PAT への切り替えを推奨

### 注意点
- review4 スキルは user スコープ（`~/.claude/skills/review4/`）と project ローカル（`.claude/skills/review4/`）の両方に存在し、project ローカルが優先される。両者を同期させた
- arxiv MCP は `uvx` で初回起動時にダウンロードが走るため、初回のみ少し時間がかかる

## review4 / kaggle-discussion-sync を auto-invocable に変更

- `disable-model-invocation: true` を削除し、`description` に明示的な TRIGGER 例文を追加
- 対象: `~/.claude/skills/review4/SKILL.md`, `.claude/skills/review4/SKILL.md`（プロジェクト同期）, `~/.claude/skills/kaggle-discussion-sync/SKILL.md`
- これでユーザーが「レビューして」「discussion を取ってきて」と自然言語で書いた際、Claude が自動でスキルを起動する
- `kaggle-writeup-fetch` は既に同じ形式（明示 TRIGGER 文）で auto-invocable だったため変更なし
